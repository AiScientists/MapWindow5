How to create a plug-in for MapWindow5

We're working on an easy workflow for developers to create their own plug-in for MapWindow5.
We don't have a lot of documentation yet. You need to learn the structure by looking at existing plug-ins.
We've made a template plug-in, which you can use as a base for your own plug-in.

This is the recommended workflow. We've tested in on a Windows10-x86 machine with VS2013 Community.

  1. Install the latest beta installer, available at mapwindow5.codeplex.com/releases and install it in a different location than C:\Program Files because you'll need rights to write your plug-in dll to it.
    I suggest C:\dev\MapWindow5\
  2. Download this zip file. It contains a VS2013 solution with several projects for a few core plug-ins and the template plug-in.
    Unzip it, I suggest C:\dev\plugins\
  3. Open the solution. You need to change some paths.
    • Select the properties of each project and add a reference path to the location of MapWindow5, see step 1.
    • In the properties of the plug-in you want to debug you also need to set the location of MapWindow.exe.
    You should now be able to compile and debug your plug-in.

If you have questions you can post them at the discussion page on codeplex: mapwindow5.codeplex.com/discussions
Happy coding and keep us informed of your progress.
